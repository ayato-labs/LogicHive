# Development Philosophy

> "The universe is written in the language of mathematics." — Galileo Galilei  
> システムの真実は数値に宿る。計測できないものは制御できない。

---

## 1. Self-Use First

本プロジェクトは**自分自身が使うために作るツール**です。  
機能の優先順位は「自分が実際に使うか」を第一基準とします。

その結果として生まれた恩恵をOSSとして公開していますが、  
スコープは常に「自分にとっての最高」を追求することに置かれています。

---

## 2. Automate Everything

「環境構築できなくて使えない」はエンジニアとして許容できない体験です。  
そのため、セットアップは**ワンコマンドで完結する**ことをコア要件とします。

```bash
# これだけで動く状態を常に維持する
run.bat   # Windows
./run.sh  # Unix
```

ユーザーが複雑な環境構築を強いられる設計は、機能の問題と同等に扱います。

---

## 3. Scope of Support

本プロジェクトは **AS-IS（現状有姿）** で提供されます。

**対応スコープ内（issueとして歓迎）:**
- 再現可能なバグ報告（環境情報・再現手順を含むもの）
- 機能提案（自分のユースケースと重なるもの）
- Pull Request

**対応スコープ外:**
- ユーザー固有のOS・ネットワーク・ハードウェア依存の問題
- 個別環境のデバッグ対応
- 手取り足取りのセットアップ支援

ローカルAIアプリはPyTorch・CUDA等の巨大ライブラリに依存する性質上、  
全環境を完璧に保証することは構造的に不可能です。  
無限のサポート対応で開発が止まることこそ、OSSにとって最大の損失です。

---

## 4. Monetization Threshold

現在のサポート方針は、以下の条件で変わります。

| トリガー | 変化する内容 |
|---|---|
| 有償版リリース | 購入者への環境サポートを提供 |
| 法人契約 | SLAを定義し、個別対応を提供 |

無料公開の現状においては、上記スコープ外の対応はありません。

---

## 5. Sustainability Principle

「俺の環境で最速・最強で動く。使いたい人は使ってください。」

このスタンスは冷淡さではなく、**個人開発を健全に継続するための設計**です。  
開発者が燃え尽きずに走り続けることが、プロジェクトの長期的な価値になります。
